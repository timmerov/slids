compiler_v2 — PLAN: Phase 2 control flow (multi-session)

The flow-control track of Phase 2 (declarations + flow). Var decls + definite
assignment + the unused-local sweep already LANDED (see plan.txt, and the
resolve-stage notes in readme.txt). This file plans the remaining flow
constructs. Dependency-driven ordering: each session's definite-assignment
"join" increment builds on the prior. Test files live in test_v2/flow/*.sl.

Locked design decisions:
  1A — switch cases are INDEPENDENT (no C-style fall-through).
  2A — unreachable-code detection lands now (a stmt after an abrupt sibling).
  3B — constant-true loops (`while (true)`) get NO special-case; every loop is
       treated as possibly-zero-iterations. Deferred to the while landing
       (Session 3): when `while (true)` + return-correctness forces it, revisit;
       until then a break-only-exit loop in a value-returning fn over-demands a
       trailing return. Note in todo.txt if a test trips it.


THE JOIN MODEL (the load-bearing design; entirely a resolve-stage concern)

  initialized_locals means "definitely assigned on EVERY path reaching this
  point." A branch splits flow; a merge INTERSECTS the incoming branches — a var
  is initialized after a join only if initialized on all paths into it.
  Straight-line (today) is the degenerate one-path case.

  Abrupt completion — the key subtlety is WHICH branches reach the join. A
  branch that returns/breaks/continues never arrives, so it must not drag the
  intersection down. Model: a lattice with empty set = bottom and the UNIVERSAL
  set = top, the "abrupt/unreachable" marker; join = intersection, top is its
  identity. resolveStmt/resolveBlock return a Completion { Normal, Abrupt }
  alongside mutating the set:
    - Normal — falls through, contributes its actual out-set to the join.
    - Abrupt — return/break/continue, contributes top, skipped in the ∩.
  after = ∩ contributions; if ALL branches are Abrupt, the construct is Abrupt
  (after-point unreachable → drives 2A unreachable detection).

  Two things that do NOT join:
    - read_locals (drives the unused sweep) is a MONOTONIC UNION — a read on any
      path is a use. No snapshot/restore at branches.
    - body_locals — declaration tracking. Under blocks, the unused sweep and the
      init-set save/restore move from per-FUNCTION to per-BLOCK (sweep a block's
      declared locals at block exit, matching pushFrame/popFrame).

  Data shape (all transient in resolve, no parse::Tree schema change):
    - enum Completion { Normal, Abrupt } returned by the body walk.
    - a loop-context stack of { break_accum, continue_accum, label } for
      break/continue targeting.
    - set snapshots are plain std::set<int> copies at branch points (id-keyed).


SESSION 1 — bare blocks (foundational)   [LANDED]

  Infrastructure-heavy, branch-light. Everything below depends on it. LANDED;
  canon test_v2/flow/block.sl (+ .exp). The probe earned its keep — resolve/
  classify looked clean but codegen had TWO real bugs only `llc` surfaced:
  (1) alloca register collided on shadowing (`%x` emitted twice) — fixed by
  suffixing the register with the entry id, `%<name>.<id>`; (2) emitFunction had
  its OWN trailing-return check that asserted on a trailing block — fixed with an
  ast-side endsInReturn mirroring classify's.
  * new node: kBlockStmt (parse + ast). Trips -Wswitch-enum across every stage —
    update all switches in one pass.
  * grammar: `{ stmts }` -> kBlockStmt; a block is legal anywhere a stmt is.
  * resolve: push/pop a frame per block. Migrate the DA discipline from
    per-function to per-block: enclosing initialized_locals + read_locals FLOW
    THROUGH (an assignment/read inside a block affects the outer local — a block
    is scoped, not isolated); only body_locals is block-scoped, save/restore it
    so the unused-local sweep runs at block exit over locals declared in THIS
    block.
    SHADOWING (canon): ALLOWED by default — an inner-block local masks an outer
    name via the innermost-first live-scope walk; the outer entry is restored on
    pop (standard block scoping). Exceptions, NEITHER reachable in bare blocks:
    (a) parse ambiguities (type-name-as-variable; class-related, deferred); (b) a
    body-TOP local shadowing a parameter — already caught, since the param shares
    the body's top frame so a same-name top-level local is a plain Duplicate
    declaration (canon negative lives in function/calls.sl, shadow_ban). A
    NESTED-block local reusing a param name is ALLOWED (B-pure: the param is
    recoverable outside that block). So Session 1 adds NO new shadow check.
  * classify: return-correctness recursion — the "last statement is a return"
    check (classify.cpp) must recurse into a trailing kBlockStmt so
    `int f(){ { return 1; } }` passes. Minimal: if the last child is a block,
    check ITS last child (recursively).
  * codegen: blocks transparent (emit children in order). Alloca register names
    are `%<name>.<entry_id>` so a shadowing inner local doesn't collide with the
    outer `%x` (the SymTab is entry-id-keyed; this naming is the shadowing-safety
    mechanism reused by every later flow construct's body). emitFunction's
    trailing-return terminator decision uses the same endsInReturn recursion.
    desugar's PPID statement-bump splicing recurses into a block (lowerStatement-
    List) so a bump inside a block splices within it, not at function scope.
  * NOTE: the Completion { Normal, Abrupt } lattice + 2A unreachable-statement
    detection are SPLIT OUT to Session 2 (if/else), where the join makes them
    load-bearing. Bare blocks are sequential/single-path; no Completion yet.
  * tests: test_v2/flow/block.sl — nested scope; shadowing ALLOWED (pos: inner
    local masks an outer name, outer restored after the block); assignment to an
    outer var from inside a block, read after (pos: init flows out); per-block
    unused sweep (pos: a block-local declared-never-read errors at block exit).
  * exit: blocks nest + scope correctly; per-block unused sweep fires; trailing
    block satisfies return-correctness.

SESSION 2 — if / else (the join) + the Completion lattice   [LANDED]

  LANDED; canon test_v2/flow/ifelse.sl (+ .exp), 8 negatives. kIfStmt
  (children[0]=cond, [1]=then-block, [2]=optional else = block or nested kIfStmt
  for `else if`); `if`/`else` already lexed → no lex/token change. resolveStmt
  now RETURNS Completion; resolveStmtList threads 2A (stmt after an Abrupt
  sibling → "Unreachable statement.", dead tail skipped) and is used by BOTH the
  function-body loop and the kBlockStmt arm. The join: snapshot S, resolve each
  arm from S, ∩ the normally-completing arms (abrupt = top/skipped, no-else = S);
  if Abrupt iff has-else and both arms Abrupt. classify: condition truthy-coerce
  (isCoercibleToBool) + endsInReturn split into per-node endsInReturnNode (both-
  arms-return if satisfies return-correctness). desugar: lowerIfStmt (condition =
  self-contained phrase, arms recurse). codegen: emitToBool + conditional br +
  then/else/merge (reuses emitLogical primitives, no phi); arm ending in return
  emits no br-to-merge, merge block OMITTED when every arm returns (2A guarantees
  nothing live follows). Audit clean (no silent defaults / mis-attribution /
  stored canonical names — join is std::set<int>). Deferred-test items in
  todo.txt: if-condition-type diagnostic + `^`-ref condition (untriggerable until
  void/`^` values exist).

  The genuinely new analysis; small surface, proves the whole model. The
  Completion { Normal, Abrupt } lattice + 2A unreachable-statement detection
  land HERE (split out of Session 1) — the join is what makes them load-bearing.
  * new node: kIfStmt.
  * introduce enum Completion { Normal, Abrupt } returned by the body walk. A
    block/stmt is Abrupt if it cannot complete normally (a return today; break/
    continue in Session 3). 2A: a stmt following an Abrupt sibling in a block ->
    "Unreachable statement." Retrofit the block walk (Session 1) to thread it.
  * classify: condition is a condition-expression (reuse the ! && || coercion
    check).
  * resolve — the join: snapshot S at entry; resolve then from S, restore,
    resolve else from S; after = contrib(then) ∩ contrib(else), abrupt branches
    contributing top. No-else falls out free (T ∩ S = S, an if-without-else never
    adds). The if is Abrupt iff both arms are.
  * codegen: conditional br + merge block (allocas, no phi).
  * tests: test_v2/flow/if.sl — both-arms-init -> assigned after (pos); one-arm
    -> use-after is a neg; return-in-one-arm -> other arm's init survives (pos);
    unreachable-after-both-return (neg).
  * exit: the ∩-with-top join correct end-to-end.

SESSION 3 — while + break/continue (unlabeled) + POST-CONDITION while   [LANDED]

  Loops pin the init-set; break/continue exercise Abrupt propagation. Scope grew
  mid-session: POST-CONDITION while (do-while) landed too (the user pulled it in),
  which is what made the break/continue accumulators load-bearing — a pre-
  condition while alone would not have needed them.
  * new nodes: kWhileStmt, kDoWhileStmt, kBreakStmt, kContinueStmt (all stages,
    one -Wswitch-enum pass). `while`/`break`/`continue` were already lexed.
  * grammar: parseWhileStmt dispatches on the token after `while` — '{' -> post-
    condition (`while { body } (cond);`), else -> pre-condition. Shared
    parseParenCondition: an EMPTY condition (`while ()` / `while {} ()`) synthesizes
    the always-true literal "1" (slids convention; if/else still owes this —
    todo.txt).
  * resolve: a Tree::loop_stack of LoopFrame { break_init/seen, continue_init/seen }
    (id-keyed, ∩-folded, top-seeded by `seen`). PRE-condition: after = S (possibly-
    zero; the zero-iteration exit dominates, so break_accum is ignored). POST-
    condition: body runs once from S -> body-out B; condition reads checked against
    B ∩ continue_accum; after = that ∩ break_accum. break/continue: Abrupt, error
    if loop_stack empty ("A 'break'/'continue' statement must be inside a loop."),
    fold the current init-set into the enclosing frame.
  * 3B: every loop possibly-zero — NO constant-true special-case; a loop is never
    a terminator (Normal), so a function ending in a while/do-while still demands a
    trailing return (negatives cover it). No test tripped the while(true)+return-
    correctness corner, so nothing added to todo.txt for it.
  * codegen: pre = head/body/exit (test-first); post = body/cond/exit (body-first).
    break->exit, continue->head (pre) / cond (post), via a LoopCtx { header, exit }
    threaded through emitStmt. Back-edge emitted only if the body falls through
    (endsTerminated = return/break/continue). TWO codegen bugs in the same class as
    the v1 loop-alloca overflow were fixed: (1) ALL local allocas hoisted to the
    entry block (collectVarDecls pre-walk) — a body-local alloca would leak per
    iteration; (2) emitLogical rewritten from alloca+load to PHI nodes — an &&/|| in
    a loop condition put its alloca in the re-executed header. Both verified in IR
    + a 1e6-iteration big_loop runs without overflow.
  * tests: test_v2/flow/while.sl — pre (counting, zero-iter, break/continue in
    branches, PPID-in-cond, nested loops, empty cond, && cond, big_loop) + post
    (body-runs-once, inits-escape, break, continue-to-test, PPID-in-post-cond,
    empty post-cond, body-local hoist, cond-reads-body-var) + nested-loop
    break/continue targeting + ||/^^ conditions. 9 negatives: break/continue
    outside a loop, unreachable-after-break/continue, pre-loop init-doesn't-escape,
    do-while break-before-assign (after-set ∩) + continue-before-assign (cond reads
    ∩), while/do-while not satisfying return-correctness.
  * exit: pre + post while, unlabeled break/continue; the loop-context accumulators
    are live (do-while consumes them).

  POST-SESSION-3 SIDE QUESTS (landed on top, not part of the original 6 sessions):
  * EMPTY CONDITION everywhere — `if ()` now uses the same shared grammar helper
    parseParenCondition as while (synthesizes the always-true "1" literal). The
    "empty = true" convention now applies to if + both while forms.
  * CONSTANT-CONDITION UNREACHABLE — classify (post-constfold, so a folded literal
    / substituted const / synthesized empty-`()` is visible; resolve's 2A is pre-
    constfold). constTruth(cond) -> True/False/NotConst; const-true if flags its
    else, const-false if flags its then, const-false while flags its body
    ("Unreachable statement." at the dead branch's first stmt; empty branch =
    nothing to flag). Const-TRUE loops are NOT flagged (3B — the unreachable
    after-code case is the todo.txt deferral); a do-while is never flagged (body
    runs once). Tests: ifelse.sl (empty-then/empty-else positives + 7 const-dead
    negatives spanning bool/int/char/float/folded/substituted/else-if) +
    while.sl (do-const-false + while-true positives confirm no false flag).

SESSION 4 — for-loops   [LONG-FORM + RANGED + ENUM + TYPELESS/REUSE LANDED; array/tuple/class pending]

  ARCHITECTURE (revised from the original range-only plan): the LONG-FORM for is
  the CANONICAL for node, and every other for shape (range `i : 0..N`, array,
  enum, tuple, class for-each) DESUGARS to it. So the long form is handled
  natively in every stage; the others become a desugar-to-kForLongStmt rewrite.

  LANDED — long-form `for (varlist) (cond) {update} {body}` (kForLongStmt):
  * node kForLongStmt, children [cond, update-block, body-block, varlist-decl*].
  * grammar parseForStmt; varlist = comma-separated TYPED decls (explicit type
    required this round; empty list ok); cond via the shared parseParenCondition
    (empty -> always-true). 3 scope frames: for-scope (varlist) > sibling
    {update, body} blocks, so the body may shadow a for-var.
  * resolve: body-then-update (execution order). cond reads from post-varlist S';
    body from S' (break/continue target the for); update checked against
    body-out ∩ continue_accum (sees body-assigned vars, not continue-skipped ones).
    possibly-zero -> after = S. UPDATE RESTRICTION: no break/continue/return in the
    update (in_for_update + for_update_floor; a loop nested in the update gives its
    own legal break/continue target; return is banned transitively).
  * codegen: varlist init stores once (allocas hoisted); head(cond)/body/update
    /exit; continue -> update (the loop var advances on continue), break -> exit.
  * tests: test_v2/flow/forlong.sl — 18 positives (basic, empty clauses/cond,
    multi-var, continue->update, break, nested, 3-frame shadow, typed-no-init,
    body-first DA, nested-loop break/continue targeting, break-in-nested-loop-in-
    update, PPID in cond + varlist-init, && cond, empty body, update-local) + 8
    negatives (update break/continue/return, transitive return in nested update
    loop, for-var uninit in cond, continue-undercuts-update-read, possibly-zero
    escape, return-correctness).

  LANDED — ranged `for (var : start .. [cmp] end [op step]) {body}` (first shape):
  * grammar dispatches on the token after the first `type var`: ':' -> ranged,
    else -> long-form varlist. Ranged DESUGARS in the grammar to a kForLongStmt
    (synthesized `_$end`/`_$step` hidden vars + cond `var cmp _$end` + update
    `var = var op step`), tagged with the `..` token (Node::range_dotdot_tok).
    cmp `< <= > >= !=` default `<`; op `+ - * / << >>` default `+1`; operands
    (start/end/step) are unary-expressions (parens for compound, no binary eaten).
    Explicit-type var this round.
  * classify EMPTY-RANGE check (ranged-for only, gated on range_dotdot_tok): if
    start + end fold to literals and `start cmp end` is false -> "Invalid range."
    caret on the `..` (rangeFirstTestFalse compares the two literals, int/float).
    NO infinite-loop guard (deferred — todo.txt literal-fold check).
  * tests: test_v2/flow/forrange.sl — 21 positives (all cmp/op variants incl.
    `/ >> << *`, descending, parens, variable/call/unary/PPID operands, float +
    char ranges, nested, break/continue, boundary) + 7 negatives (empty range via
    `< > != <= >=`, boundary, float).

  LANDED — enum `for (var : Enum) {body}` (second shape):
  * grammar: the ':' branch parses an operand; '..' -> ranged, a bare identifier ->
    parseEnumFor building a kForEnumStmt {loop-var, enum-ref, body}.
  * resolve REWRITES it in place to a kForLongStmt over the enum's first..last
    DEFINED members (first/last kConst by id order in the enum's ns_frame),
    referenced by qualified name (`Enum:first` / `Enum:last`) so resolve/constfold
    fill the values; cond `var <= _$end`, update `var = var + 1`; tagged range-
    derived so a descending/empty enum (first > last) trips the empty-range
    "Invalid range." check (caret on the enum name). Value-based, no contiguousness
    check. Errors: unknown enum / not-an-enum / no-members. Never survives resolve,
    so downstream stages carry assert-unreachable cases only.
  * tests: test_v2/flow/forenum.sl — 13 positives (count/sum members, int + enum +
    char loop-var types, non-zero-start, non-contiguous (value-based, hits non-
    members), single-member, negative-valued, namespace-qualified enum, nested,
    break/continue, empty body) + 4 negatives (descending->Invalid range, not-an-
    enum, unknown enum, zero-member enum).

  LANDED — TYPELESS varlist + enclosing-scope REUSE (rides inferred-init):
  * grammar: parseForVarHead parses `[type] name` for every varlist entry (first +
    each comma) and both colon-form heads. TYPELESS when the lead is neither a
    primitive type-start nor a qualified typed-decl shape (isTypeStart ||
    looksLikeQualifiedTypedDecl) — then the lead identifier is the var name,
    return_type left empty. ('A:B name' vs typeless 'name : range/enum' disambiguate
    structurally, no naming-convention pivot.)
  * resolve: the for-varlist loop intercepts a typeless decl (empty return_type,
    non-const, unqualified). WITH init -> reclassified kAssignStmt, routed through
    the existing kAssignStmt path: an in-scope name REASSIGNS the enclosing local
    (reuse — no fresh alloca), an unknown name becomes a fresh inferred-init local.
    NO init -> reuse an in-scope local as a no-op slot (neutralized to an empty
    block); a name that resolves to a non-local errors ("Cannot use <kind> 'x' as
    a loop variable."), a truly-undeclared name errors ("Cannot infer the type of
    'x'; it has no initializer." + a placeholder entry so the cond/update/body
    reads don't cascade). The after-set is S' (post-varlist, captured before the
    body) — the varlist inits run once unconditionally so they ESCAPE, making a
    reused var observable after the loop (the v1.md chess-sample reuse fix); a fresh
    for-var is in S' but out of scope (inert, ids unique). LOOP-VAR REUSE RULE: a
    local of the loop-var name already in scope is REASSIGNED, not freshly
    allocated. `int row = -1; for (row = 0)(row<n){++row;}{}` leaves row == n.
  * classify: a ranged/enum for (range_dotdot_tok set) with a typeless (inferred)
    loop var stamps the loop var's resolved type onto the synthesized typeless
    `_$end`/`_$step` (children[4..]) before classifying them, so their bounds flex
    into it — matching an explicitly-typed range. Reuse type-compat is option A:
    the normal assignment conversion rejects an incompatible reuse (e.g. an int
    `row` over a `0.0..1.0` range -> "Cannot implicitly convert 'float' to 'int'").
  * codegen / desugar: NO change — a reused varlist slot is a kAssignStmt (stores
    into the existing alloca, collectVarDecls skips it) and a fresh one a
    kVarDeclStmt (hoisted alloca); both already flow through.
  * tests: added to the natural homes — forlong.sl (typeless fresh / reuse /
    no-init reuse / mixed varlist / shadow / PPID; negatives: no-init undeclared,
    no-init wrong-kind const+function, reuse-const, fresh loop-var use-after-loop),
    forrange.sl (typeless range / reuse / char / cmp+step; negatives: empty range,
    int64-narrowing), forenum.sl (typeless count/sum / char / qualified; negative:
    descending). REUSE: outer local reassigned, reads the last value after.
  * exit: for-loops desugar to the canonical form and run; loop-var reuse correct.

SESSION 5 — switch / case (1A)

  * new node: kSwitchStmt (+ case labels).
  * 1A independent cases: each case branches from S, no fall-through. after = ∩
    of normally-completing cases (abrupt skipped); defaultless switch contributes
    S (can't add — same shape as no-else).
  * resolve/classify: case values are constants of the switch type (constfold
    already folds); duplicate-case detection.
  * codegen: branch chain or jump table.
  * tests: test_v2/flow/switch.sl — all-cases-init+default -> assigned after;
    defaultless -> not; duplicate-case (neg); break semantics.
  * exit: switch with the independent-case join.

SESSION 6 — labels (labeled break/continue)

  Final flow piece; everything it targets now exists.
  * grammar: labels on loops/switch; `break label;` / `continue label;`.
  * resolve: label scope; resolve a label to its enclosing loop/switch via the
    loop-context stack (now carrying labels); error on unknown/duplicate label.
  * codegen: branch to the labeled exit/continue block.
  * tests: test_v2/flow/labels.sl — labeled break from nested loop, labeled
    continue, unknown-label (neg).
  * exit: Phase 2 flow complete.


CROSS-CUTTING (every session)

  * Build discipline: each new Kind -> update all stage switches in one pass;
    verify make RC=0 + the g++ relink before running a binary (a failed
    -Wswitch-enum arm leaves a stale slidsc).
  * read_locals never joins — monotonic union across all paths; only
    initialized_locals snapshots/intersects.
  * Doc cadence: at each session close, mark the plan.txt Phase 2 item LANDED +
    update the resolve-stage paragraph in readme.txt. The join model gets one
    memory at Session 2 (when it's real), extended per-construct.
  * Inferred-init (`x = 42`, no type) is an orphan Phase-2 item NOT on this flow
    track — grammar + resolve/classify, no flow interaction. NEXT UP (resume
    point): preceded by a minimal `##type(x)` test instrument, then inferred-init
    (which also unlocks the for-loop typeless/reuse loop var — Session-4 remainder).
    See plan.txt (Phase 2 var-decl line) + todo.txt (##type item).

SEQUENCING

  1->2->3 is strict (lattice -> join -> loop-context). 4 and 5 both depend on 3;
  4 before 5 only because for-loop desugar is heavier and de-risks the
  canonical-form approach early. 6 is last (needs loops AND switch to target).
