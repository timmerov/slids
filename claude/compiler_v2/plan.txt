compiler_v2 — PLAN (the main quest)

Feature implementation order. Items are open or LANDED; updated as we make
progress. Outstanding bugs / reach goals / deferred side-quests live in
todo.txt, not here. Detailed per-stage current-state notes (what each stage
does today, stage by stage) live in readme.txt.

Each phase adds primitives the next phase depends on. widen.cpp threads through
every conversion site as features land; treat each new expr/decl/assign site
as a widen call-out.

PIPELINE — current shape

    lex -> numeric -> grammar -> resolve -> constfold -> classify
        -> desugar -> optimize -> codegen

  resolve owns: parse::Entry vector population, frame push/pop, ident
  resolution (resolved_entry_id stamping), wrong-kind diagnostics
  (assign-to-function, call-a-variable), duplicate-decl + signature
  mismatch + arity-mismatch + multi-arg-print-intrinsic detection,
  type-spelling validation (widen::isKnownType for declared / return /
  parameter types), namespaces + enums (lowered to entries here), and the
  orphan-function check (declared-but-never-defined). Two passes per scope
  (collect decls then walk bodies) so forward references work.

  classify is type inference + (Phase 3) overload resolution. Stamps
  inferred_type / op_type, rejects no-common-type binaries, non-coercible
  operands for `! && || ^^` (and the `&&= ||= ^^=` aug-assigns), an if / while /
  for condition not coercible to bool, non-numeric
  shift sides, bitwise on float, and return correctness (non-void must end in a
  return — recursing into a trailing block and a both-arms-return if/else; void
  can't return a value). Reads entries via resolved_entry_id but never mutates
  the symbol table or pushes frames.

Phase 6 — const correctness
  Threads through phases 3-5; land after classes have the bones.

  QUALIFIER + MUNGE + CASTS LANDED (the SYNTAX / type-representation half; NO
  enforcement). const is now a tracked TypeRef qualifier — Form::kConst, a facet
  wrapping `underlying`, TRANSPARENT for matching (strip/deepStrip/classify see
  through it) but VISIBLE in spell(). Placement encodes deep vs shallow: an OUTER
  kConst is `const T^` (deep — the whole pointer + its data); a kConst pointee is
  `(const T)^` (shallow). Helpers: internConst (collapses const-const), removeConst
  (drops every const, keeps structure/aliases), deepConst (the MIRROR — const-
  qualifies every mutable position: array elem, tuple slot, pointer + pointee).
  intern() peels a leading `const ` FIRST (so the prefix binds loosest); spell()
  parenthesizes a const child under a `^`/`[]`/`[N]` suffix so the shallow form
  round-trips.
  * pass-by-ref-to-const munge (mungeParamType): a non-`mutable` reference /
    iterator param's pointee becomes const (`T^`->`(const T)^`, `T[]`->`(const T)[]`),
    an array param -> pointer to a const array; primitive by value untouched; the
    synthesized `_$recv` receiver skipped (const methods below). `##type` shows it.
  * `mutable` — a pointer/iterator PARAMETER qualifier (opts out of the munge);
    rejected on a non-pointer param, a class field, and any non-parameter position.
  * `<const>` / `<mutable>` qualifier casts + full-type `<const T^>` / `<T^>` —
    add / remove const on a pointer (value-preserving; const erased in IR).
  * CONST VARIABLES (non-scalar leading `const`): a leading `const` on an array /
    tuple / class / pointer / iterator LOCAL is a not-mutable VARIABLE (allocated +
    initialized), not a substituted constant — resolve's constNeedsStorage gate
    routes it to a kLocalVar with a deepConst-wrapped type; a foldable SCALAR const
    stays substituted. A non-scalar const at a NON-RUNTIME scope (file / namespace /
    class body) is a global -> "requires global storage" (Phase 8). ast `is_const`
    now means "substituted constant" (set from the entry kind in desugar). A TYPELESS
    const (`const x = rhs`) is the same split decided in CLASSIFY: constfold defers
    the unfoldable case, classify infers the rhs type and routes a non-scalar one to
    the same kLocalVar + deepConst path (a leftover typeless scalar errors; the
    runtime-scalar const variable is a todo). Canon: test_v2/misc/mutable.sl,
    assign/infer.sl.

  ENFORCEMENT — STILL THIS PHASE (none landed; const is transparent for now, so
  the const->mutable direction is silently accepted):
  * `const T` lvalue carries const, rvalue read is mutable (the value-category half)
  * method-level const (`name(params) const { ... }`); const self (receiver munge)
  * const-overload matcher (canonical-type strips "const" for match-key)
  * const->mutable rejection at call sites (without an explicit `<mutable>`)
  * write-through-const rejection at every assign-family site
  * `<const>` cast becomes load-bearing (the only way to mint a const-value rvalue)
  * `mutable` becomes REQUIRED on a move/swap operator param (`op<--`/`op<-->`,
    once operator overloading + those operators exist — Phase 5+)
  * goal B (publish-side via transport `.slt` -> `.slh`) — design-level only

  KNOWN LIMITATIONS (const): a leading `const` needs an initializer. On a foldable
  SCALAR it is the named-CONSTANT marker (substituted); a SCALAR const-qualified
  VARIABLE is written in a type position (`(const int)^ r`), since a leading
  `const int x` IS the named constant. At a NAMED declarator a top-level sized dim
  goes on the name, never inline — `(const int)[3] a` is rejected, the same as
  `int^[3] v`; in a NAMELESS type position the dim is inline (`alias V =

  const VARIABLE via classify (LANDED — see the inferred-type-constants note above);
