compiler_v2 — PLAN (the main quest)

Feature implementation order. Items are open or LANDED; updated as we make
progress. Outstanding bugs / reach goals / deferred side-quests live in
todo.txt, not here. Detailed per-stage current-state notes (what each stage
does today, stage by stage) live in readme.txt.

Each phase adds primitives the next phase depends on. widen.cpp threads through
every conversion site as features land; treat each new expr/decl/assign site
as a widen call-out.

PIPELINE — current shape

    lex -> numeric -> grammar -> resolve -> constfold -> classify
        -> desugar -> optimize -> layout -> codegen

  resolve owns: parse::Entry vector population, frame push/pop, ident
  resolution (resolved_entry_id stamping), wrong-kind diagnostics
  (assign-to-function, call-a-variable), duplicate-decl + signature
  mismatch + arity-mismatch + multi-arg-print-intrinsic detection,
  type-spelling validation (widen::isKnownType for declared / return /
  parameter types), namespaces + enums (lowered to entries here), and the
  orphan-function check (declared-but-never-defined). Two passes per scope
  (collect decls then walk bodies) so forward references work.

  classify is type inference + (Phase 3) overload resolution. Stamps
  inferred_type / op_type, rejects no-common-type binaries, non-coercible
  operands for `! && || ^^`, an if condition not coercible to bool, non-numeric
  shift sides, bitwise on float, and return correctness (non-void must end in a
  return — recursing into a trailing block and a both-arms-return if/else; void
  can't return a value). Reads entries via resolved_entry_id but never mutates
  the symbol table or pushes frames.

Phase 2 — declarations + flow   <-- IN PROGRESS
  * var decls — typed init + declare-then-assign LANDED; inferred init (no
    type spelling) LANDED. inferred-init = `x = expr` with no type: resolve's
    kAssignStmt arm, on a truly-undeclared name (!isQualified && resolveName < 0),
    creates a kLocalVar with empty slids_type, rewrites the node to a kVarDeclStmt,
    resolves the rhs (so `x = x` reads x uninitialized), marks it initialized;
    classify's kVarDeclStmt arm then infers the rhs, NORMALIZES a literal-inferred
    type to its preferred spelling (preferredSpelling: int32->int, uint32->uint,
    float32->float; a typed rhs keeps its spelling), inherits rhs.alias_label, and
    WRITES BACK entry.slids_type (the one deliberate classify symbol-table
    mutation, assert-guarded). Aug-assign on an inferred var fixed: classify's
    kAugAssignStmt arm RE-READS entryType into s.return_type (resolve cached it
    empty before the inferred decl was stamped). Canon: expression/inferred.sl.
    The for-loop typeless/reuse loop var (Session-4 remainder + the v1 chess-sample
    reuse fix) is now WIRED IN on this mechanism (see the flow item below).
    Definite assignment LANDED: a local read before
    written is an error ("Use of uninitialized variable"); params seeded
    initialized. Unused-local sweep LANDED: a body-declared local never read
    errors ("Unused local variable" if never written, else "set but never
    used"); consts + params exempt. Straight-line only — control-flow joins
    attach per-construct as the branches below land. Canon: expression/vardecl.sl.
  * flow constructs (blocks, if/else, while, for, switch, labeled break/
    continue) — a 6-session track in plan-flow.txt (the join model + per-construct
    landing stages; tests in test_v2/flow/). Decisions locked: switch cases
    FALL THROUGH C-style (the original "independent cases" plan was wrong — canon
    in switch.sl/slids_reference.md), unreachable-code detection on, no
    constant-true loop special-case.
    Sessions 1 (bare blocks) + 2 (if/else) + 3 (while + do-while + break/continue)
    LANDED. 1: kBlockStmt, per-block scope + DA migration, shadowing allowed,
    alloca names entry-id-suffixed for shadow safety, trailing-block return-
    correctness. 2: kIfStmt + the Completion { Normal, Abrupt } lattice (resolveStmt
    returns it; resolveStmtList threads 2A unreachable-statement detection), the
    ∩-at-merge definite-assignment join (abrupt arm = top, else-less = S),
    condition truthy-coercion, codegen conditional-br/merge (merge omitted when
    both arms return), endsInReturnNode return-correctness. 3: kWhileStmt (pre,
    after = S, possibly-zero) + kDoWhileStmt (post `while {body} (cond);`, body
    runs once so inits escape — after = (body-out ∩ continue_accum) ∩ break_accum)
    + kBreakStmt / kContinueStmt (Abrupt, loop-only, fold into a Tree::loop_stack
    of break/continue accumulators); empty condition = always-true literal (a
    slids convention; grammar-synthesized); ALL local allocas hoisted to the entry
    block (v1 stack-overflow fix) + emitLogical rewritten to phi nodes (was alloca,
    leaked per-iteration in a loop condition); LoopCtx-threaded break/continue
    codegen; endsTerminated (return/break/continue) for br decisions. Side quests
    on top of Session 3: empty condition (`if ()` too, via the shared
    parseParenCondition); constant-condition unreachable detection (classify,
    post-constfold) — const-true if flags the else, const-false if/while flags
    then/body ("Unreachable statement."); const-true loops + do-while never flagged
    (3B). Session 4 (for) IN PROGRESS — the LONG-FORM for landed
    (kForLongStmt `for (varlist) (cond) {update} {body}`, the canonical for node
    that the other for shapes will desugar to): explicit-type varlist, 3 scope
    frames (for/update/body, body may shadow a for-var), body-then-update DA
    (update reads body-out ∩ continue_accum), possibly-zero (after = S), update
    may not break/continue/return, continue -> update in codegen. The RANGED for
    also landed (`for (var : start .. [cmp] end [op step]) {body}`, cmp
    `< <= > >= !=` default `<`, op `+ - * / << >>` default `+1`, operands are
    unary-expressions): desugars in the grammar to a kForLongStmt with synthesized
    `_$end`/`_$step`; explicit-type var; empty-range compile error ("Invalid
    range." on `..`) when start+end are constant and start-cmp-end is false; no
    infinite-loop guard (todo.txt). The ENUM for landed too (`for (var : Enum)
    {body}`): a kForEnumStmt that resolve rewrites in place into a kForLongStmt
    over the enum's first..last DEFINED members (members referenced by qualified
    name; value-based, no contiguousness check); a descending/empty enum reuses
    the empty-range "Invalid range." check. The TYPELESS / REUSE loop var landed
    across all three live forms (parseForVarHead disambiguates `[type] name` vs a
    bare typeless name; resolve's varlist loop routes a typeless+init decl through
    the kAssignStmt path = reuse-an-enclosing-local or fresh inferred, a typeless
    no-init decl reuses-or-errors, and the after-set is S' so a reused var is
    observable after; classify propagates the loop-var type onto a typeless ranged
    `_$end`/`_$step`; reuse requires a compatible type, else the normal assignment
    conversion rejects it). FOR-ARRAY landed in Phase 4 (`for (var : arr)`, a
    resolve-stage desugar to kForLongStmt — see Phase 4); for-tuple/for-class
    still pending tuples/classes. Session 5 (switch) LANDED — kSwitchStmt +
    kCaseClause, C-style FALL-THROUGH, integer-class scrutinee, unique in-range
    integer-constant labels (incl. qualified `case Dir:N:` via a terminator-aware
    parse), default singular + anywhere, naked break exits the switch / naked
    continue skips it to the enclosing loop, exhaustive switch is a
    return-terminator (containsBreak agreed across classify+codegen), lowered to
    an `llvm switch`. Session 6 (labels) LANDED — loop labels (`:name` after the
    body; default keyword names for/while), numbered (`break N;` Nth loop, skipping
    switches) + named (`break name;`) break/continue, resolved to a hop count and
    walked over a LoopCtx linked list in codegen; no break/continue of any flavor
    in a for-update; labels on loops only (named-switch-break distant-future).
    Phase 2 FLOW COMPLETE; for-array landed in Phase 4, for-tuple/class pending.
  * return — LANDED (kReturnStmt + return-correctness check from Phase 1).

Phase 4 — types beyond primitives
  * references (`T^`, no arithmetic)                                LANDED
    (kAddrOfExpr `^var` -> reference, kDerefExpr `v^` deref as lvalue+rvalue,
    kNullptrLiteral -> anyptr; `^` lexes as kBitXor, disambiguated structurally:
    prefix=address-of, postfix=deref [unless the next token starts a primary,
    then it's XOR], type-suffix=reference. kStoreStmt is the general lvalue-
    expression store [`ref^ = v`]. Compare-only `== !=` [v1 rule]; ordering and
    all arithmetic rejected. Both lower to LLVM `ptr`.)
  * iterators (`T[]`, additive arithmetic)                          LANDED
    (`iter^` deref, `iter[i]` subscript, `^arr[i]` -> iterator [operand-driven:
    var->`^`, indexed->`[]`]; `iter ± int` GEP by element stride, `iter - iter`
    -> intptr [byte diff / sizeof elem], `++`/`--` step one element; full six
    comparisons. iterator->reference assignment demotes [v1]. `char[]` from a
    string literal.)
  * fixed-size arrays (`T name[N]`, multi-dim)                      LANDED
    (decl size after the NAME; multi-dim REVERSED — `int[3][5]` is 5 chunks of
    3 = LLVM `[5 x [3 x i32]]`, leftmost dim innermost; `arr[i]...` subscript
    [every dimension required] via one reversed-index GEP; `^arr[i]` -> iterator;
    constant-index [incl. char] bounds-checked; NO initializer lists [tuples
    out]. Whole-array definite-assignment is a monotonic may-set [a subscript
    write assigns the array; read-before-any-write errors]; arrays join the
    unused-local sweep.)
  * for-array (`for (var : arr)`)                                  LANDED
    (resolve-stage desugar to kForLongStmt, like for-enum: synth `_$idx$<tok>`
    walks 0..len-1, the loop var bound at the body top — by reference [`T^ v`],
    by value [`T v`], or typeless [by value; reuse-an-enclosing-local-or-fresh].
    One-dimensional only; a by-ref var's base type must equal the element type.)
  * memory mgmt (`new T(init)`, `new T[n]`, `delete ptr`)
  * anonymous tuples (`(T1, T2, ...)`)
  * type aliases — LANDED (value form `alias Name = type;`, file + function
    scope, transparent; resolveTypeSpelling substitutes the chain; desugar drops
    kAliasDecl). Deferred alias items (##type, function-rename, templated) → todo.txt.
  * qualified type names — LANDED. A type spelling may be namespace-qualified
    (`Space:Dir`, `::A:B:T`): grammar's parseType accepts a qualified name and a
    looksLikeQualifiedTypedDecl lookahead routes `Space:Dir x` to a var-decl;
    resolveTypeSpelling resolves a `:`-bearing spelling through the shared ns
    chain walker to the underlying before any downstream stage. Diagnostics:
    not-a-namespace / not-a-namespace-member / not-a-type-in.
  * constants — LANDED (file + function scope; kConst entries with captured
    values; constfold substitution + iteration + fixpoint check; sharp
    diagnostics for assign-to-const, range overflow, missing init, dup, cyclic).
    Inferred-type constants LANDED (typeless `const name = expr`, grammar infers
    when `const` is followed by `name =`): the strong/weak distinction via
    strong_type on nodes / const_strong_type on entries — constfold's combineStrong
    + makeLitAt propagate strength through folds (a strong/typed operand makes the
    result strong + takes its type, both-strong uses commonType, a bool/comparison
    result is weak); tryCaptureConst infers a typeless const (strong rhs takes its
    type, a bare-literal rhs is WEAK -> weakDefaultType preferred spelling). NOT in
    this landing: strong-const WIDENING enforcement in runtime contexts (todo.txt).
  * namespaces — LANDED (pulled forward from Phase 8). `Name { members }` open/
    reopen at file or function-body scope; members are consts, functions, nested
    namespaces; kNamespace entry with persistent frame identity; open-namespace-
    chain bare lookup (qualifiers optional, `::` defeats a shadow); qualified
    `A:B:C` / `::`; inline qualified member decls; bare `alias Ns;` import;
    desugar hoists member functions with entry-id-derived symbols (no cached
    canonical names). Members may also be enums (registerNamespaceTree registers
    type-introducing members first, for order-independence). Cross-TU namespaces
    + `.slh` propagation → Phase 8.
  * enums — LANDED. `enum [type] [Name] ( m1 [= v], ... )`, type-first (default
    int), optional name. Lowered at RESOLVE: named → a kNamespace whose
    slids_type carries the underlying (transparent type) + kConst members;
    anonymous → bare kConst members in the enclosing frame. May be declared at
    file, function, or NAMESPACE scope (registerEnum takes a parent_ns; a named
    enum becomes a member of its enclosing namespace, an anonymous one's members
    do). Auto-increment from 0, C rules (implicit member =
    clone(last-explicit-init) + offset, folded by constfold). Ambiguous-bare-ref
    diagnostic. Member inits resolve in a late pass (after all file-scope entries
    exist) with the enum's own frame open, so a member init can reference a
    file-scope const or a sibling member bare. A named enum's members carry the
    enum name as a `##type` label (registerEnumMembers stamps each member's
    alias_label; anonymous enums have no name -> bare `const int`), and a const's
    stored value takes its DECLARED type's literal kind (tryCaptureConst →
    literalKindForType), so char/bool consts and `enum char`/`enum bool` members
    print as their declared type rather than the int-folded initializer. Canon:
    test_v2/constfold/enum.sl. Deferred enum items (reach-goal drop-qualifier,
    reopen, char-arith-loses-char-ness bare-expr case) → todo.txt.

Phase 5 — classes
  * class definitions (fields, ctor, dtor, methods)
  * inferred field types (`Foo(name = const_expr)`)
  * operator overloading per slid
  * move / swap (`<--`, `<-->`)
  * destructor balance invariant
  * for-each protocol (op[]+size OR begin/end/next)
  * stringification operators (`##name`, `##type`, `#`) — the 7 `##` macros
    (##file / ##line / ##func / ##name(x) / ##date / ##time / ##type(x)) LANDED,
    pulled forward as the inferred-types test instrument (##type(x) -> a new
    kStringifyType Kind that classify lowers to a kStringLiteral). ##type reports
    an alias/enum-VARIABLE label (parallel alias_label channel) + the const
    qualifier (a bare const ident -> `const <T>`, expression-read strips const).
    ##type also resolves a bare/qualified TYPE NAME to its underlying — an alias
    (chain fully resolved), an enum name, or a qualified enum type (`Space:Dir`)
    -> the underlying spelling (`##type(Integer)` -> `int`, `##type(E)` -> `int`,
    `##type(Space:Dir)` -> `int`), DISTINCT from a VALUE of that type which keeps
    its label (`Integer x; ##type(x)` -> `Integer`). resolve's kStringifyType arm
    dispatches on the operand entry kind (a TYPE -> resolveTypeSpelling onto
    return_type; a VALUE -> the value path; neither -> a diagnostic), classify
    short-circuits when return_type is stamped. An enum MEMBER reports
    `const Enum` (`##type(Dir:kNorth)` -> `const Dir`, via each named-enum
    member's alias_label). The `#x` 5-tuple REMAINS (Phase 5, blocked on tuples
    + `^`); aliased-FUNCTION `##type(Fn)` waits on function aliases (todo.txt).
    Canon: test_v2/misc/macros.sl, alias.sl, constfold/enum.sl.

Phase 6 — const correctness
  Threads through phases 3-5; land after classes have the bones.
  NOTE: `##type`-level const-qualifier REPORTING already landed (a bare const
  ident reads `const <T>`, an expression-read strips const) as a side-effect of
  ##type + inferred constants — but that is reporting-only (v2 still models const
  as an entry KIND, not a tracked type qualifier). ENFORCEMENT below is this phase.
  * `const T` qualifier; lvalue carries const, rvalue read is mutable
  * method-level const (`name(params) const { ... }`); const self
  * pass-by-ref-to-const default for `T^`/`T[]` params; `mutable` opts out
  * const-overload matcher (canonical-type strips "const" for match-key)
  * const→mutable rejection at call sites
  * write-through-const rejection at every assign-family site
  * `<const>` cast — the only way to mint a const-value rvalue
  * goal B (publish-side via transport `.slt` → `.slh`) — design-level only
