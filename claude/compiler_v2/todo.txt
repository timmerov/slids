compiler_v2

PLAN — feature implementation order

Each phase adds primitives the next phase depends on. widen.cpp threads through
every conversion site as features land; treat each new expr/decl/assign site
as a widen call-out.

Phase 0 — foundations (largely landed)
  * lex stage + import wrapper
  * numeric stage — literal validation + canonicalization (mostly landed;
    int/uint/float/char/bool all flow lex→numeric; items 5 and 7 of the
    migration still open — see project_v2_numeric_stage memory)
  * constfold stage — between grammar and classify (TODO; spec in
    test_v2/parse/widen.sl and test_v2/const/fold.sl):
      - rule 4 (binary on two literals): widen both to common computational
        type int64/uint64/float64, perform, assign nominal type by value
      - rule 3 (unary on literal): widen to computational type, perform;
        -uint and -bool lowered to -1 * literal then rule 4; ~ on bool error
      - shift on two literals: rule 4 with shift count >= bitwidth → 0;
        negative shift count → compile error
      - algebraic identity simplifications: x+0 → x, x*1 → x, x*0 → 0,
        x/1 → x, x%1 → 0, x|0 → x, x&0 → 0, x^0 → x, x<<0 → x, x>>0 → x,
        x|~0 → ~0, x&~0 → x, 0-x → -x
      - logical with constant: x && false → false, x || true → true,
        x && true → x, x || false → x, x ^^ false → x, x ^^ true → !x
        (safe today; needs side-effect check once function calls land)
      - division/mod by zero (literal divisor) → compile error
      - every literal gets a nominal type assigned (parse::Node field):
        bool / char / int8/16/32/64 / uint8/16/32/64 / float32/64 by value
  * diagnostic sink + source rendering
  * single-source error pipeline (main.cpp early-out between stages)
  * build hygiene (-Wall -Werror -Wswitch-enum)
  * widen primitives table + literal range checks

Phase 1 — expression core
  * unary operators (!, ~, full -)
  * binary operators (arithmetic, comparison, bitwise, logical, shift)
  * parenthesized expressions, precedence + associativity
  * calls (zero-arg, one-arg, multi-arg)
  * compound assigns (+=, -=, *=, /=, %=, &=, |=, ^=, <<=, >>=, ||=, &&=)
  * PPID — pre/post-inc/dec via phrases
  * (literal+literal int fold moves to constfold per Phase 0)

Phase 2 — declarations + flow
  * var decls (typed init, inferred init, declare-then-assign)
  * blocks
  * if / else, while
  * for-loop (six shapes desugar to one canonical form)
  * switch / case
  * labeled break / continue
  * return

Phase 3 — functions
  * definitions: params (typed + inferred), return type
  * declarations (signature-only)
  * overload resolution (arity → types → const)
  * nested functions + capture analysis
  * method dispatch (`.method()`)
  * function pointers

Phase 4 — types beyond primitives
  * pointers (`^` reference, no arithmetic)
  * iterators (`[]`, arithmetic allowed)
  * memory mgmt (`new T(init)`, `new T[n]`, `delete ptr`)
  * anonymous tuples (`(T1, T2, ...)`)
  * type aliases (`alias X = T;`, `alias X<...> = T;`)
  * constants
  * enums

Phase 5 — classes
  * class definitions (fields, ctor, dtor, methods)
  * inferred field types (`Foo(name = const_expr)`)
  * operator overloading per slid
  * move / swap (`<--`, `<-->`)
  * destructor balance invariant
  * for-each protocol (op[]+size OR begin/end/next)
  * stringification operators (`##name`, `##type`, `#`)

Phase 6 — const correctness
  Threads through phases 3-5; land after classes have the bones.
  * `const T` qualifier; lvalue carries const, rvalue read is mutable
  * method-level const (`name(params) const { ... }`); const self
  * pass-by-ref-to-const default for `T^`/`T[]` params; `mutable` opts out
  * const-overload matcher (canonical-type strips "const" for match-key)
  * const→mutable rejection at call sites
  * write-through-const rejection at every assign-family site
  * `<const>` cast — the only way to mint a const-value rvalue
  * goal B (publish-side via transport `.slt` → `.slh`) — design-level only

Phase 7 — class advanced
  * reopens; incomplete-class semantics
  * hoisted classes (file-scope hoist across reopens)
  * derived classes
  * virtual methods + virtual destructors
  * inheritance contracts (`= default`, `= delete`)
  * `instantiation` statement / unnamed instances (`Type(args);`, `Type;`)

Phase 8 — namespaces + cross-TU
  * namespaces (`Name {}`)
  * .slh → .sl propagation (header → consumer, transitive impl-load)
  * C interop (`= import`, `import {}`)
  * globals (lazy ctor/dtor, sentinel + ensure widget, dtor walker, `global;`)

Phase 9 — templates
  * template functions
  * template classes (plain, derived, with nested members)
  * post-compile-pre-link stage (`--instantiate` aggregation)


KNOWN GAPS — current state

GRAMMAR
  * <type> recognizes all built-in primitives and <T>[] for any of them.
    Table-driven via kPrimitives in grammar.cpp.
    - no other array shapes (T[N], multi-dim, etc.)
    - no class / struct / enum / alias type names yet
  * parameter lists not parsed (only bare `()`)
  * expression parsing: literal leaves (string, int, char, bool, float)
    + bare identifier + unary `! ~ + -` + full binary set (arith / bitwise /
    shift / comparison / logical) with C precedence + parenthesized grouping.
    - unary `+ - ~ !` on literals fold at desugar
    - no postfix ops (field access, indexing, call-through, `^` deref);
      slot into parsePostfix as their phases land
    - no function calls inside expressions
    - no `<Type^>` or `(Type=expr)` casts
    - float literals must be lossless decimal (no hex form yet, no inexact
      values like 1.1 which would require LLVM hex encoding); codegen
      normalizes `1e40`-style text to `1.0e40` so LLVM accepts the float form

WIDENING + LITERAL RANGE CHECKING (widen.h / widen.cpp)
  * widen::checkIntLiteralFits / checkFloatLiteralFits enforce per-type
    literal ranges. Int literal at float dest must be exactly representable
    in the float's significand. Float literal at int dest must be integer-
    valued and in range.
  * widen::convert handles var-to-var conversions:
      - signed → strictly larger signed (sext)
      - unsigned → strictly larger unsigned (zext)
      - unsigned → strictly larger signed (zext, value-preserving)
      - bool → any integer (zext from i1) / any float (uitofp from i1)
      - float → strictly larger float (fpext)
      - int → float when the int type's range fits the float's significand
        (sitofp / uitofp). int8/int16/uint8/uint16 → float32; up to int32/
        uint32 → float64; int64/uint64 → no silent float.
    Rejects (sentence-shaped messages):
      - any same-or-narrower target ("Cannot implicitly narrow ...")
      - signed → unsigned ("(signed → unsigned)")
      - unsigned → same-width signed ("(unsigned to same-width signed)")
      - int → float when not value-preserving ("Cannot implicitly convert ...")
      - float → int: never silent ("Cannot implicitly convert ...")
  * widen::commonType — "smallest type large enough to hold either operand".
    Used by codegen's binary-op emitter for arith / bitwise / comparison.
    Returns false when no type fits (uint64 + any signed; int64 + float64).
  * widen::intLiteralFits / floatLiteralFits — silent variants of the
    literal checks. Used by codegen's literal-flex rule (try-fit-into-other-
    operand-type first; fall back to default literal type when not).
  * codegen handles binary-op shapes:
      - arith / bitwise / comparison: commonType → emit at common → convert
        result to dest_type. Comparison result is bool, converted to dest.
        Float arith covers `+ - * / %` (frem for `%`); float bitwise (& | ^)
        rejects sharply: `Bitwise '&' not defined on floating-point type '...'`.
      - shift: LHS dictates result type; RHS resized to LHS width via
        trunc/zext (LLVM's same-operand-width requirement). Float LHS
        rejects sharply: `Shift '<<' not defined on floating-point type '...'`.
      - logical (&& || ^^): each operand → bool independently; result bool.
      - literal + literal int/char: compile-time fold via tryFoldIntBinary,
        produces an untyped literal text that flexes to the surrounding target.
  * codegen's literal-flex uses defaultLiteralType for the literal's
    fallback type (int32 by magnitude → int64 → uint64; bool/char/float as-is).
  * Source attribution is plumbed: every widen call takes (file_id, tok)
    and threads into diagnostic::report.
  * codegen's SymTab carries slids_type alongside llvm_type so widen has
    the typed source for ident-load conversions.
  * Still TODO:
      - explicit type conversion syntax `(Type=expr)` not yet parsed
      - explicit cast syntax `<Type^>` (Phase 4)
      - widen at function-call arg sites (no params yet)
  * call statements: exactly one argument
    - zero-arg calls not parsed
    - multi-arg calls not parsed
  * top-level: only function definitions and function declarations
    - imports work (lex-time wrapper expands them; grammar accepts declarations
      from imported .slh files)
    - no classes, namespaces, enums, aliases, globals
    - function signatures are name + bare `()` only — no params, no overloads
    - no duplicate-declaration check (same name twice silently dedup'd by
      codegen skipping FunctionDecl nodes; will need real symbol resolution)
  * statement-level: var-decl, assign, augmented-assign (desugared to assign),
    call, return
    - no if/else, no for/while, no block

DESUGAR
  * copies parse → ast and propagates `(file_id, tok)` per node so codegen
    errors attribute to source.
  * tryFoldUnary — constant-folds `+ - ~ !` applied to a literal operand.
    `+`/`-`/`~` walk the literal text (stripLeadingPlus / toggleSign /
    foldBitNotIntText); `!` produces a fresh kBoolLiteral. Fresh nodes inherit
    the unary expr's (file_id, tok).
  * tryDesugarAugAssign — rewrites `lhs op= rhs;` → `lhs = lhs op rhs;`.
    Lvalue is a bare ident today; complex lvalues (when arrays / fields land)
    need a bind-to-tmp step here to avoid double-eval.
  * Binary literal+literal fold currently lives in codegen (tryFoldIntBinary
    inside emitBinary) — the natural home is desugar (paired with
    tryFoldUnary; recursive folds for `1+2+3` fall out for free). Pending.
  * Other canonicalization not yet here: PPID lowering, for-loop shapes
    → canonical, operator-overload dispatch lowering, stringify lowering,
    etc. — slot in as their phases land.

NUMERIC
  * stage landed; handles kIntLiteral, kUintLiteral, kCharLiteral,
    kFloatLiteral, kBoolLiteral. Per-kind validation + canonicalization with
    sentence-shaped diagnostics ("Hex literal overflows uint64.", etc.).
  * lex now emits source-form text per kind; numeric canonicalizes:
    - kIntLiteral: text in decimal digits → pos checked → overflow / malformed
    - kUintLiteral: text "0xFF" or "0b1010" → parsed → decimal text out;
      kIntLiteral whose value > INT64_MAX flips kind to kUintLiteral here
    - kCharLiteral: in-quotes span → escape interpreted → decimal byte value
    - kFloatLiteral: source minus underscores → strtod → %.17g canonical
    - kBoolLiteral: "true"/"false" → "1"/"0"
  * Still open from the 1-7 migration table:
    - item 5: widen::checkUintLiteralFits + checkCharLiteralFits for proper
      diagnostic attribution ("Integer literal" → "Unsigned literal" /
      "Character literal") at codegen.cpp:496-514 routing sites
    - item 7: codegen float32 hex bit-pattern emit. `float xf = 3.14;`
      currently rejected by llc because slidsc emits decimal for float32
      where llc requires the 16-hex-digit form. Computation: source decimal
      → double → (float) cast → re-promote to double → uint64 bit pattern →
      emit as `0x` + 16 hex digits.

NEW SILENT DEFAULTS — from items 3+4+6 (kUintLiteral resurrection)
  * desugar.cpp tryFoldUnary: `-` on kUintLiteral calls toggleSign on text
    but doesn't transition kind. Result: kUintLiteral with negative text
    (semantically incoherent). Per the H-rule (-uint = -1 * uint, becomes
    signed via mixed widening), should set child->kind = kIntLiteral after
    toggleSign, plus a magnitude check (parse uint64; if mag > 2^63 →
    compile error, since negation underflows int64).
  * codegen.cpp exprType for kUintLiteral unconditionally returns "uint"
    (uint32). Large kUintLiteral (value > UINT32_MAX) gets typed as uint32
    — silent narrowing. Should call defaultLiteralType (or share a helper)
    to pick uint32 vs uint64 by value. Surfaces today only via __println
    segment-typing (print.cpp routes via exprType); larger surface as
    expressions land.
  * codegen.cpp defaultLiteralType parse-failure fallbacks (`return
    "uint64"` / `return neg ? "int64" : "uint64"`) silently swallow
    malformed text. Numeric is the precondition guarantor; the fallback
    arm should be assert(false && "defaultLiteralType: malformed text from
    numeric") so a numeric bug surfaces as a crash, not a wrong type.

CONSTFOLD
  * stage doesn't exist yet. Slots between grammar and classify per the
    project_compiler_redesign 9-stage pipeline. Spec in widen.sl rules
    1-4, 8-9, plus identity simplifications and division-by-zero detection.
    See "Phase 0" entry above for the full TODO list.

CLASSIFY
  * no-op; no symbol resolution, no type inference, no overload resolution

OPTIMIZE / LAYOUT
  * no-op; no perf rewrites, no name mangling, no field-offset computation

CODEGEN
  * VarDecl alloca'd at use site, not at function entry (works for now since
    no control flow; will matter once if/for show up)
  * AssignStmt: only direct identifier lvalues (no field, no index)
  * llvmTypeFor handles all primitives + <T>[] (lowers to ptr); class /
    struct / enum / alias not yet
  * bool lowers to i1 (per redesign-memory intent; v1 uses i32 for bool)
  * return value coerced to fn_return_type via emitExpr's widen path;
    non-widenable returns produce a widen diagnostic at the return stmt
  * target triple hard-coded to x86_64-pc-linux-gnu

ERROR REPORTING
  * source attribution + caret-sled rendering landed. parse::Node and
    ast::Node carry (file_id, tok); grammar stamps at construction; desugar
    propagates; widen / codegen / print all thread the relevant node's
    position into every diagnostic::report call. Exception: lex's
    "cannot open root_path" fires before any source is read (no token
    exists) — falls back to bare `slidsc: error: <msg>`.
  * parser error messages are minimal ("expected '...'"); no suggestions, no notes

TEST COVERAGE GAPS — features implemented in v2 with no test exercising them
  * Adjacent string-literal concatenation (`"a" "b"` → `"ab"`).
    parsePrimary's kStringLiteral arm has a while-loop that concatenates
    adjacent tokens. No test uses two adjacent string literals.
  * Logical short-circuit observable verification — deferred until
    expressions have side effects (call expressions / PPID). emitLogical
    emits the conditional branches but nothing in any test would distinguish
    short-circuit from full evaluation today.

PRINT INTRINSIC (print.h / print.cpp)
  * handles __println / __print with a single arg only:
    - string literal  -> printf(@.str_N)
    - bare identifier -> load + printf("%s\n", val); assumes the ident's
                         llvm-type is `ptr` (i.e. char[]); other types
                         produce a printf("%s", non-ptr) at runtime which
                         will misinterpret the value
  * no `+` concat chains
  * no slice / non-literal segments
  * no multi-arg
  * shares strings::Pool with the rest of codegen; depends on codegen::SymTab
    via codegen.h. To delete: remove print.* + the two call sites in
    codegen.cpp + the SRCS entry in Makefile.
