                   diagnostic::Sink& diag)
  in classify.cpp, defined AFTER the classify* / check* helpers and the
  isArrayFromTuple family, just BEFORE classifyStmt. It is the full canonical chain
  that is currently INLINED at decl-init and assign:

    if (dest == kNoType) return;
    if      (isArrayFromTuple(dest, rhs))       classifyArrayFromTuple(tree,dest,rhs,diag);
    else if (isArrayFromTupleValue(dest, rhs))  classifyArrayFromTupleValue(dest,rhs,diag);
    else if (isTupleFromArrayValue(dest, rhs))  classifyTupleFromArrayValue(dest,rhs,diag);
    else if (checkArrayValueAssign(dest, rhs, diag)) { /* array value -- handled */ }
    else { checkPtrAssign(...); checkStrongConstAssign(...); checkSlidAssign(...); }

  inferExpr is ALREADY called at every site BEFORE the checks — the helper is the
  post-inference chain ONLY. Do NOT put inferExpr inside it.

CODEGEN (the seams the helper exposes)

  - checkArrayValueAssign EXACT-MATCH -> PER-ELEMENT. The relation (readme
    TUPLE/ARRAY) is shape-match + each element re-enters the matrix
    (`int8[4] -> int[4]` should work). Today checkArrayValueAssign rejects unequal
    deepStrip. Change: classify checks per-element compatibility; codegen lowers an
    aggregate assign to per-element converts (widen::convert per slot/element). A
    whole-aggregate insertvalue/memcpy copy survives only as the all-identical
    degenerate case.
  - kDestructureStmt codegen (~1719) already does extractvalue + store per slot;
    make each store run widen::convert so a slot may widen into a wider target.
  - END-STATE (optional, bigger): the numeric cross-family reject lives in
    widen::convert (codegen) + the literal-fit checks, NOT in classify. The POINTER
    rule already gates in classify and asserts in convert; make the NUMERIC rule
    match — classify's helper becomes the single authority and widen::convert drops
    to pure lowering + asserts (no diagnostics, no double-report).
