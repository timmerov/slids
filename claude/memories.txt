project_slids_overview.md
========================================
Slids is a compiled, systems-level programming language. Source files (.sl) compile to LLVM IR (.ll) via slidsc (written in C++17), then to native binaries via llc + g++.

Status: Active development. Compiler is working with a substantial feature set.

Why: User is building a new language with intentional design trade-offs distinct from C++.
How to apply: Use these decisions when suggesting features, syntax, or implementation approaches — don't suggest features that conflict with these choices.

KEY LANGUAGE DECISIONS
- Everything is a "slid" — one unified construct for functions, classes, methods, constructors
- ^ = reference (no arithmetic); [] = iterator (arithmetic allowed)
- () signifies data; {} indicates code — consistent throughout
- .slh files are declarations only — no code, no template bodies
- No lambdas, no recursive functions
- int always 32-bit; intptr is pointer-sized signed (not size_t); char is unsigned
- delete nullifies pointer after freeing; <- is explicit pointer theft (move)
- sizeof returns signed intptr (not unsigned size_t — avoids underflow bugs)
- (type=expr) for explicit type conversion; incomplete classes via ... prefix/suffix

IMPLEMENTED FEATURES (each has a sample file as regression test)
- Basic class, constructor/destructor, methods, slices, copy_chars (string01-string05)
- External method definitions, block-style external methods, IndexAssignStmt (string06-07)
- Method overloading by type-suffix mangling (string08)
- Operator overloading with sret calling convention; forward declarations with ; (string09)
- Pointer-as-array indexing; pointer arithmetic; move/memcpy semantics (string10-14)
- Free function overloads with mangled names (string15-16)
- String literal + char concatenation; int-to-string conversion; uint arithmetic (string17-18)
- sizeof(type), sizeof(expr), obj.sizeof(), TypeName.sizeof() — returns intptr; compiler-generated __sizeof function for every complete slid (sizeof.sl)
- Strong type inference: x = 0; infers int; hex/binary → uint; digit separators (type_infer.sl)
- (type=expr) conversion syntax; class conversion (Value=42); chained (float32=int=3.14) (typeconv.sl)
- Pointer reinterpretation casting (cast.sl)
- Swap operator <-> (SwapStmt) (string.sl reverse method)
- Templates with explicit type args add<int> (template1.sl)
- Templates with inferred type args; int widening; slid-over-primitive; reference type errors (template2.sl)
- Incomplete classes via Foo(...) declaration and Foo(..., fields) implementation; __pinit chains to __ctor; __sizeof enables consumer allocation without knowing private fields
- -MF/-M dependency file generation for Makefile auto-dep tracking
- -M flag: Program.imported_headers in parser, written to .d file in make-compatible format
- new/delete for user-defined types (new_delete.sl)
- new T[N] stores hidden 8-byte count header; delete T[] reads count, reverse dtor loop, frees header ptr

TEMPLATE INSTANTIATION ACROSS TUs
- Template bodies in .sl files (never .slh); impl file found by naming convention
- Local templates: always define inline
- Imported templates: emit declare + write .sli file
- slidsc --instantiate <dir>: reads all *.sli, deduplicates, writes __instantiations.sl
- instantiate foo<T>; top-level keyword forces inline define
- Import-once guard via shared_ptr<set<string>>
- pending_declares_ deferred to module scope

LINKAGE MODEL
- Functions/methods in .slh → external linkage (define)
- Functions/methods only in .sl → define internal
- main → always external
- Nested functions → always define internal
- exported_symbols_ set tracks what gets external linkage

FORWARD DECLARATIONS OF CLASS METHODS
- void Simple:hello(); (colon-style) — body=null, codegen skips
- Simple { void goodbye(); } (block-style) — same null-body treatment

TEMPLATE CLASS SUPPORT
- Vector<T>(fields) { methods } syntax
- scanForSlidTemplateUses() pre-pass; instantiateSlidTemplate() in codegen_template.cpp
- Naked block { } is valid statement with RAII scoping

TRANSPORT REMOVAL + __SIZEOF
- transport keyword and --export-path flag removed entirely
- __sizeof compiler-generated for every locally-complete slid
- sizeof(var) parser fix: isSizeofTypeName checks uppercase initial letter

RANGED FOR LOOPS (session 2026-04-21)
- for x in (1, 2, 3) — ForTupleStmt: flat array alloca + index; stacksave/stackrestore per iteration
- for ch in "Hello" / for x in arr — ForArrayStmt: string literal iterates len-1 chars (skipping null); fixed-size array T[N] uses GEP into [N x llvmType(T)]
- for a in lp (class with begin()/end()) — same ForArrayStmt AST node; codegen detects slid_info_.count(arr_type), emits begin()/end() calls, counter-increments loop var each iteration
- All three for-in forms: new loop variable scoped to loop (erased from locals_/local_types_ at end)
- Parser: after 'in', kStringLiteral or kIdentifier → ForArrayStmt; '(' → range/tuple path
- ForArrayStmt added to all 9 scan sites in codegen.cpp; collectStringConstants also calls collectExpr on array_expr

COMPILER ERROR DETECTION
- Duplicate ctor _() or dtor ~() in a slid body → compile error

REMAINING TEMPLATE WORK
- @impl "other" annotation in .slh to override same-name impl convention
- Name conflict detection for same mangled name from two headers
- Emit alwaysinline/inlinehint on template instantiations

MAKEFILE STRUCTURE
- bugs/ — bug regression tests
- test/ — language feature tests
- sample/ — sample programs
- work/ — multi-file development area: string, templates, vector

CODEGEN INTERNALS
- sret calling convention for functions returning slid types
- dtor_vars_ cleared at start of each function/method emit
- pending_temp_dtors_ flushed before dtor_vars_ in emitDtors()
- exprLlvmType(VarExpr): slid type names return "ptr" not "i32"
- pinit rule: ALL struct initialization through __pinit — never skip it

FILE LAYOUT
compiler/ (slidsc C++17 source), sample/ (regression tests), work/ (multi-file dev area), bin/ (binaries), build/ (intermediates)

project_nested_slids.md
========================================
Nested slid classes — flat hoist semantics (sample/nested.sl, regression test).

Why: User wanted `Inner` defined inside `Outer { ... }` for organization, accessible by `Outer:Inner` outside Outer and bare `Inner` inside Outer's methods. Inner has zero access to Outer's fields/methods — purely a syntactic nesting.
How to apply: When extending nested-slid behavior, treat `Outer.Inner` as a top-level slid; it has no implicit relationship to Outer at codegen time.

DESIGN
- Source syntax: `:` separator (`Outer:Inner`); internal/AST canonical name uses `.` (`Outer.Inner`) — `.` is legal in LLVM IR identifiers, `:` is not (would need quoting).
- Inner is hoisted to `program.slids` after parse; codegen sees a flat list.
- Inside Outer's methods, parser maintains `nested_alias_` map ("Inner" → "Outer.Inner"); applied at parseTypeName after consuming the base identifier.
- Outside Outer, parseTypeName consumes `:Identifier` after the base when the next token is not `(` (disambiguates from external-method def `Outer:method(`).
- isVarDeclLookahead extended to skip `:Identifier` between base and var name.
- No duplicate-name check (an early version had one, but it broke incomplete classes — see GOTCHAS).

PARSER ENTRY POINTS (parser.cpp)
- SlidDef.nested_slids (parser.h)
- Parser.nested_alias_ map (parser.h)
- parseTypeName: qualified-name + alias substitution
- parseSlidDef body loop: isNestedSlidDecl lambda + recursive call, with save/restore of current_slid_fields_ and nested_alias_
- Parser::parse(): hoist pass at end. parent_name MUST be passed by value into the recursive lambda — push_back into program.slids reallocates and would dangle a reference.

NESTED TEMPLATES
- `InTemplate<T> { ... }` inside Outer works: hoist preserves type_params; alias substitutes `InTemplate` → `Outer.InTemplate` before parseTypeName's template-arg loop mangles to `Outer.InTemplate__T`.
- Nested template body is cloned at instantiation via cloneBlock/cloneExpr in codegen_template.cpp.

GOTCHAS (encountered during impl)
- Hoist lambda captured parent_name by reference into a SlidDef inside program.slids; vector reallocation during push_back invalidated it. Symptom: cryptic `basic_string::_M_create` or `std::bad_alloc` only when ≥2 nested slids in the same outer. Fix: pass parent_name by value.
- A duplicate-name check on program.slids breaks incomplete classes. Incomplete classes legitimately produce two SlidDef entries with the same name: declaration with has_ellipsis_suffix=true, implementation with has_ellipsis_prefix/is_transport_impl=true. Codegen depends on the two-entry encoding (codegen.cpp:467, 470, 776, 798, 805, 886, 1233). Don't add such a check.
- cloneExpr in codegen_template.cpp does NOT handle StringifyExpr — `##type(m_)` etc. Putting one inside a template method body causes "cloneExpr: unhandled expression type" on instantiation. Pre-existing gap, not nested-classes-specific.

OUT OF SCOPE (v1)
- Inner as a field type of Outer (fields parse before nested defs become visible).
- `Outer:Inner make()` as a top-level function return type — currently misparsed as external-method def at line ~2026 (`Outer:Inner` matches the external-method shape).
- Class-template ctor-arg type inference: `Outer:InTemplate main_it(11);` (no `<...>`) does NOT infer T from the ctor arg. Must write `Outer:InTemplate<int> main_it(11);`. Function templates have inference; class templates don't.
- Exporting nested types via .slh.
- Deeper nesting works via recursion in the hoist pass but isn't otherwise exercised.

KEYWORD GOTCHA
- `in` is `kIn` (lexer.cpp:161); cannot be used as a variable name. To allow `Inner in(5);` the lexer would need to demote `in` to a contextual keyword (drop kIn, have for-loop parser check kIdentifier with value "in").
