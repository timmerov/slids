
new feature: stringification the # operator.
# turns things into string literals.

variables:
int some_var = 42;
##name(some_var) -> "some_var"
##type(some_var) -> "int"
#some_var -> tuple: ("int", "some_var", 42)

c++ parity:
##line -> "12" current line in the file
##file -> "stringification.txt"
##func-> "foo"
##date -> "Apr 22 2026"
##time -> "21:52:02"

maybe in the future:

##pathname -> "/home/timmer/Documents/slids/stringification.txt"
##function_mangled -> "String__foo"

enums:
a bit of a reach goal.
this one needs runtime code.
even if it's just looking up the name in a table.
we don't know in advance the value of piece.
enum Piece { kKing }
Piece piece = kKing;
##value(kKing) -> "kKing"
##value(piece) -> "kKing"

bools:
same issues as enums.
but easier.
bool yes = true;
bool no = false;
##value(true) -> "true";
##value(no) -> "false";
