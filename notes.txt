
for loops:
kinda hate the current syntax.
    for iter in container { }
'in' should not be a keyword.
possibilities:
    for (int x : 0..10) { }
    for (x : start..end) { }
    for (x : enum) { }
    for (iter : container) { }
long form for example:
    for (
        int i = size,
        char[] src = src_buffer,
        char[] dst = dst_buffer
    ) (i >= 0) {
        --i; ++src; ++dst;
    } {
        dst^ = src^;
    }
middle form?:
    for (int x=0; x<10; ++x) { }

do we need the op^[] ?
no.
it returns a reference to the indexed value.
T^ op^[](index) {
    // cast pointer [] to reference ^ is automagic.
    T^ ref = storage_ + index;
    // alternatively:
    T^ ref = begin() + index;
    return ref;
}
how would we use it?
these are probably impossible to parse as intended.
T^ ref = ^vec[3];
T^ ref = vec^[3];
maybe just api.
T^ ref = vec.at(3);
