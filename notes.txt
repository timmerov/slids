const correctness:
objects are passed by reference.
the current implementation is mutable.
but the intention is for the default to be const.
we will need a way to make a reference mutable.
void strcpy(mutable char[] dst, char[] src);
self is always mutable.
we might want to be able to make self const.
notes on this are in future directory.

namespaces:
somewhat implemented.
    Math(
        /* constant must have initializer. */
        float pi_ = 3.14
    ) {
        /* you cannot instantiate a Math object. */
        _() = delete;

        float sin(float a);
        float cos(float a);
    }
    x = Math.sin(Math.pi_);
claude suggested using : instead of .
    x = Math:sin(Math.pi_);
Math.sin() means instantiate a temp Math and call the sin method.
which won't work because the ctor is deleted.
this feature depends on knowing that the member functions do not
access self at all.
which is similar to the problem of classifying methods as const
or mutable.

do we need the op^[] ?
no.
it returns a reference to the indexed value.
T^ op^[](index) {
    // cast pointer [] to reference ^ is automagic.
    T^ ref = storage_ + index;
    // alternatively:
    T^ ref = begin() + index;
    return ref;
}
how would we use it?
these are probably impossible to parse as intended.
T^ ref = ^vec[3];
T^ ref = vec^[3];
maybe just api.
T^ ref = vec.at(3);
