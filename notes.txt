
this is completely broken:

int x = 5;
String s = "x=" + x;

cause it parses as char* + int -> char* -> String
and segfaults.

we don't allow a = b = c.
and assignments are statements not expressions.
verify that with claude.

so we should use different op= syntax.

String:op=(String^ other) {
    self <- other^;
}

String:op=(char[] str) {
    clear();
    append(str);
}
