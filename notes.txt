globals with threads:
we probably have a race condition with globals in the default
shared memory space.
if two threads access the same global at the same time
they're both going to try to register the dtor.
which could corrupt the registry.
probably need to lock it.
globals in thread memory space are unaffected.

globals with threads:
global.self name(field list) { ctor dtor methods }
allocates memory from the thread's heap.
no locking needed.
every thread has its own copy of the global.
...
global.default name(field list) { dtor dtor methods }
allocates memory from the shared head.
locking reauired.
all threads share one copy of the global.

memory allocation with threads:
global.self is the default.
global.self allocates memory from the thread's private heap.
    ptr = new.self buffer[100];
    ptr = new buffer[100];
allocates memory from the main thread heap.
locking required:
    ptr = new.default buffer[100];
